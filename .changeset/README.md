# Changesets

Every pull request that changes a published package or bundled UI must include a changeset.

Published packages:

- `eve-studio`
- `@eve-studio/extension`

Bundled private package:

- `@eve-studio/ui` ships inside `eve-studio`; UI changes must select `eve-studio`.

Private example:

- `demo-agent`

Use:

```sh
pnpm changeset
```

For docs-only, tests-only, CI-only, or refactor changes that should not publish a package, create an empty changeset:

```sh
pnpm changeset --empty
```

Agents must not manually edit package versions or changelog sections. Versions and changelogs are generated by the release workflow's Changesets version PR.
