# Security policy

## Supported versions

Security fixes are made against the latest published versions of eve-studio
and its capture extension. Please update before reporting a problem that may
already have been fixed.

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability.

Use [GitHub private vulnerability reporting](https://github.com/multiplehats/eve-studio/security/advisories/new)
when possible. If that is not available, email
[hi@chrisjayden.com](mailto:hi@chrisjayden.com).

Include the affected version, a minimal reproduction, and the impact you
observed. You should receive an acknowledgement within three business days.
We will coordinate disclosure with you after a fix is available.
